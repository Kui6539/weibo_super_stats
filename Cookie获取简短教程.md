# 微博超话周报工具使用说明

这份说明用于第一次打开网页时快速了解各项设置。按顺序完成“抓取设置”里的内容后，点击“开始抓取并导出”即可运行。

## 抓取设置

### 超话链接或 ID

- 可以填写完整超话链接，例如 `https://weibo.com/p/100808.../super_index`。
- 也可以只填写超话 ID，例如 `100808...`。
- 如果链接无法解析，任务会在启动前提示格式错误。

### 微博 Cookie

- Cookie 用来让工具以已登录状态访问微博页面和接口。
- 推荐先使用“打开调试 Edge”和“自动获取 Cookie”。
- 如果自动获取失败，再使用手动复制方式。
- Cookie 等同于登录凭据，请不要发给他人或提交到 GitHub。

### 开始时间与结束时间

- 工具只统计这个时间窗口内发布的帖子。
- 推荐按周报周期填写，例如周六 04:00 到下周六 04:00。
- 结束时间必须晚于开始时间。

### 最大页数

- 控制最多向后翻多少页超话帖子。
- 数值越大，覆盖范围越深，但抓取时间也越长。
- 如果时间窗口比较早，可以适当增大。

### 评论系数

- 用于调节评论在热度评分中的占比。
- 默认值 `1.0` 适合大多数情况。
- 不建议低于 `0.5`。

### 请求间隔

- 每次翻页之间的等待时间，单位是秒。
- 默认 `1.0` 更稳妥。
- 如果访问过快导致接口异常，可以调大。

### 导出目录

- 生成的表格、Markdown、DOCX、图片都会放在这里。
- 如果填写相对路径，会以项目目录为基准。
- 每次任务会在导出目录下新建一个时间戳文件夹。

## 运行流程

1. 填写超话链接或 ID。
2. 获取并填入微博 Cookie。
3. 确认开始时间、结束时间、最大页数、评论系数、请求间隔和导出目录。
4. 点击“开始抓取并导出”。
5. 等待抓取完成后，在候选列表里人工选择入选帖子。
6. 确认后工具会下载图片并生成导出文件。
7. 导出完成后可预览 Markdown，也可以打开文件所在位置。

## 自动获取 Cookie

这是推荐方式，适合不想手动复制请求头的情况。

1. 点击“打开调试 Edge”。
2. 在新打开的 Edge 窗口里登录 `https://weibo.com`。
3. 登录完成后不要手动关闭这个调试 Edge 窗口。
4. 回到工具页面，点击“自动获取 Cookie”。
5. 成功后 Cookie 会自动填入文本框。
6. 如果工具成功连接调试 Edge，获取完成后会自动关闭调试 Edge。

如果自动获取失败：

- 确认调试 Edge 里打开的是 `weibo.com`，并且账号已经登录。
- 确认不是只登录了 `m.weibo.cn` 移动端页面。
- 重新点击“打开调试 Edge”，登录后再点“自动获取 Cookie”。
- 如果仍失败，改用手动复制方式。

## 手动获取 Cookie

手动复制最稳定，适合自动获取失败时使用。

1. 在常用浏览器中打开 `https://weibo.com` 并登录。
2. 按 `F12` 打开开发者工具。
3. 切换到“网络”或 “Network”。
4. 刷新微博页面。
5. 点击任意一个 `weibo.com/ajax/...` 请求。
6. 在“请求标头”或 “Request Headers” 中找到 `Cookie`。
7. 复制完整 Cookie 字符串。
8. 回到工具页面，粘贴到“微博 Cookie”文本框。
9. 点击“识别粘贴内容”，让工具自动提取 Cookie。

也可以复制浏览器里的 “Copy as cURL” 内容，然后粘贴到 Cookie 文本框，再点击“识别粘贴内容”。

## Cookie 复制注意事项

- 要复制完整一整串 Cookie，不要只复制 `SUB`、`SCF` 或 `XSRF-TOKEN` 中的某一个。
- `SUB` 通常是微博 Web 登录态的关键字段。
- `ALF` 更像登录有效期标记，不能单独代表已经登录。
- 不要复制访客态 Cookie，否则抓取时可能出现访客验证页面。
- Cookie 会过期，失效后重新获取即可。

## 常见问题

### 提示访客验证或未登录

- Cookie 可能是访客态、移动端 Cookie 或已经失效。
- 建议重新登录 `weibo.com` 后重新获取 Cookie。

### 自动获取 Cookie 成功但抓取仍失败

- 检查 Cookie 文本框里是否有 `SUB=`。
- 如果没有 `SUB=`，说明不是有效 Web 登录态。
- 改用手动复制 `weibo.com/ajax/...` 请求里的 Cookie。

### 导出结果不显示

- 如果导出目录被手动删除，工具不会继续显示旧结果。
- 重新运行一次任务即可生成新的导出目录。

### 图片下载较慢

- 图片下载会按帖子并行处理。
- 如果网络不稳定，部分图片可能下载失败，但不会阻止后续导出。

## 安全提醒

- Cookie 等同于登录态，请勿发给陌生人。
- 不要把 `weibo_stats_config.json` 或包含 Cookie 的截图提交到 GitHub。
- 如果怀疑 Cookie 泄露，请退出微博登录并重新登录。
