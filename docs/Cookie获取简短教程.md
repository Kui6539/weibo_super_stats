# 微博 Cookie 获取与使用说明

这份说明会在第一次进入网页时弹出，也可以随时点击页面右上角“使用说明”查看。工具只在本机 `127.0.0.1` 使用，Cookie 只用于访问你已经登录的微博网页状态。

## 先理解 Cookie

- Cookie 等同于微博网页版登录态，请不要发给他人。
- 不要把 `weibo_stats_config.json`、Cookie 截图或包含 Cookie 的日志提交到 GitHub。
- 有效 Web 登录态通常应包含 `SUB=`，常见还会包含 `SCF=`、`SUBP=`、`WBPSESS=`、`XSRF-TOKEN=` 等字段。
- `ALF` 更像登录有效期标记，不能单独证明 Cookie 可用。
- 访客态 Cookie 也可能包含一些字段，但抓取时容易进入“访客验证页面”。

## 推荐方式：调试浏览器自动读取

页面 Cookie 区域的“推荐方式”卡片右上角有 `Edge / Chrome` 切换项。

1. 先选择要使用的浏览器，默认是 `Edge`。
2. 点击“打开调试浏览器”。
3. 在新打开的调试浏览器窗口里登录 `https://weibo.com`。
4. 登录完成后不要手动关闭这个调试窗口。
5. 回到工具页面，点击“自动读取 Cookie”。
6. 成功后 Cookie 会自动填入，并保存到本地配置。
7. 工具成功读取后会尝试自动关闭对应的调试浏览器。

说明：

- Edge 使用独立调试目录 `.edge_cdp_profile` 和端口 `9222`。
- Chrome 使用独立调试目录 `.chrome_cdp_profile` 和端口 `9223`。
- 这两个调试窗口和你平时使用的浏览器配置分开，第一次打开时可能需要重新登录微博。
- 自动读取使用 CDP 调试协议读取 `weibo.com` 的 Cookie，不需要直接读取被浏览器锁住的 Cookie 数据库。

## 测试 Cookie

填入或自动读取 Cookie 后，建议点击“测试 Cookie”。

可能结果：

- `Cookie 可用`：可以继续预检查和抓取。
- `Cookie 可能失效`：请求没有明确失败，但登录态不够确定，建议重新获取。
- `微博返回访客验证`：大概率是访客态、移动端 Cookie 或登录态失效。
- `网络错误`：检查网络、代理或稍后重试。

测试失败时，页面不会显示完整 Cookie，也不会把完整 Cookie 写入日志。

## 手动方式：复制请求头或 cURL

如果自动读取失败，手动复制通常最稳。

1. 在常用浏览器中打开 `https://weibo.com` 并登录。
2. 按 `F12` 打开开发者工具。
3. 切到“网络 / Network”。
4. 刷新微博页面。
5. 点击一个 `weibo.com/ajax/...` 请求。
6. 在“请求标头 / Request Headers”中找到 `Cookie`。
7. 复制完整 Cookie 字符串。
8. 回到工具页面，点击“展开编辑 Cookie”，粘贴进去。
9. 点击“识别粘贴内容”。
10. 再点击“测试 Cookie”确认是否可用。

也可以右键网络请求，复制 `Copy as cURL`，把整段 cURL 粘贴到 Cookie 文本框，再点“识别粘贴内容”。工具会自动提取其中的 Cookie。

## 读取剪贴板

如果你已经把 Cookie、请求头或 cURL 复制到剪贴板，可以直接点击“读取剪贴板”。之后建议再点“识别粘贴内容”和“测试 Cookie”。

## 清空 Cookie

点击“清空 Cookie”会：

- 清空页面里的 Cookie 文本框。
- 清空本地配置文件里的 Cookie。
- 保留超话链接、时间、导出目录、主题、浏览器选择等其他配置。

如果你准备上传项目到 GitHub，建议先确认 `weibo_stats_config.json` 没有被提交。

## 常见问题

### 自动读取成功，但抓取提示访客验证

可能原因：

- 调试浏览器里没有真正登录 `weibo.com`。
- 只登录了 `m.weibo.cn` 移动端页面。
- Cookie 是访客态或已经失效。
- Cookie 中没有 `SUB=`。

建议：

- 在调试浏览器中打开 `https://weibo.com`，确认能看到自己的账号。
- 重新点击“自动读取 Cookie”。
- 如果仍失败，改用手动复制 `weibo.com/ajax/...` 请求里的 Cookie。

### 点击“打开调试浏览器”失败

可能原因：

- 没有安装所选浏览器。
- 浏览器安装路径不常见，工具没有找到。
- 端口被其他程序占用。

建议：

- 在 `Edge / Chrome` 之间切换后重试。
- 关闭旧的调试浏览器窗口后再点一次。
- 改用手动复制 Cookie。

### 自动读取本机 Cookie 失败

浏览器运行时会锁住本地 Cookie 数据库，Windows 还会对 Cookie 做加密，所以直接读取本机存储并不总是可靠。推荐优先使用“打开调试浏览器 + 自动读取 Cookie”的方式。

### Cookie 很快失效

微博可能会刷新登录态、触发验证或过期。重新登录微博网页后重新获取 Cookie 即可。

## 安全提醒

- Cookie 只应保存在你自己的电脑上。
- 不要把 Cookie 发给任何人。
- 不要把包含 Cookie 的配置、日志、截图、浏览器请求头提交到 GitHub。
- 如果怀疑泄露，退出微博登录并重新登录，让旧 Cookie 失效。
